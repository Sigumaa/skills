# ユーザー相談ゲート（yes/no）

## 判定手順
1. 下表を上から順に判定する。
2. 1つでも yes ならユーザー相談へ遷移する。
3. 全て no ならPM判断で進行してよい。

| gate_id | yes/no質問 | yes時アクション | no時アクション |
|---|---|---|---|
| DG-01 | 仕様、優先度、期限の変更が必要か | ユーザーに選択肢と推奨案を提示する | 次ゲートへ進む |
| DG-02 | 既存運用を破壊するブートストラップ更新が必要か | 変更範囲と影響を添えて承認を取る | 次ゲートへ進む |
| DG-03 | 外部契約、コスト、法務/セキュリティ判断が必要か | 前提条件を明示して相談する | 次ゲートへ進む |
| DG-04 | 失敗復旧を2サイクル実施しても解消しないか | 復旧履歴を添えて方針判断を依頼する | 次ゲートへ進む |
| DG-05 | 品質ゲート例外（skip/一時緩和）が必要か | 例外理由と期限を提案して承認を取る | 次ゲートへ進む |
| DG-06 | 依存解消のために並列計画を大幅に直列化する必要があるか | 納期影響を提示して相談する | 次ゲートへ進む |
| DG-07 | `.codex/PROPOSED_PLAN.md` に未確定事項、暗黙知、根拠不足が残るか | 不足箇所と選択肢を提示し、計画確定を依頼する | 次ゲートへ進む |

## 相談メッセージ固定項目
- 該当 `gate_id`
- 現状
- 選択肢（2-3案）
- 推奨案
- 影響範囲（スコープ/納期/品質）
