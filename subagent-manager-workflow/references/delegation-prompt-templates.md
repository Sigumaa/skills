# 委譲プロンプト固定テンプレート

## 共通ルール
- PMはテンプレートの必須項目を欠かさず記入する。
- 依頼ごとに `task_id` と `depends_on` を明記する。
- 禁止事項は省略せず、担当外作業を明確にする。

## 実装担当テンプレート
```text
[実装委譲]
- task_id:
- depends_on:
- 目的:
- 入力資料:
- 変更対象:
- 完了条件:
- 禁止事項:
  - レビュー判定を行わない
  - テスト結果を合格判定しない
  - コミットしない
- 返却物:
  - 変更ファイル一覧
  - 実装内容要約
  - 残課題
```

## レビュー担当テンプレート
```text
[レビュー委譲]
- task_id:
- depends_on:
- 目的:
- 入力資料:
- 観点:
- 完了条件:
- 禁止事項:
  - 新規実装を行わない
  - 指摘未反映のままpass判定しない
  - コミットしない
- 返却物:
  - 指摘一覧（severityつき）
  - must fix / optional fix の分類
  - 判定（pass/fail）
```

## テスト担当テンプレート
```text
[テスト委譲]
- task_id:
- depends_on:
- 目的:
- 入力資料:
- 実行範囲:
- 完了条件:
- 禁止事項:
  - 機能仕様を独断で変更しない
  - テスト失敗を黙殺しない
  - コミットしない
- 返却物:
  - 実行コマンド
  - 結果ログ（pass/fail）
  - 失敗時の再現手順
```

## コミット担当テンプレート
```text
[コミット委譲]
- task_id:
- depends_on:
- 目的:
- 入力資料:
- 対象ファイル:
- 完了条件:
- 禁止事項:
  - 指定外ファイルを混在させない
  - メッセージ規約を逸脱しない
  - pushしない（指示がある場合を除く）
- 返却物:
  - コミットID
  - コミットメッセージ
  - `git show --name-only --oneline -n 1` の要約
```
